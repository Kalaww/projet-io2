[HTML - PHP]

Il y a 2 structures HTML distincts pour notre projet:
  • celle de la page index.php : 3 colonnes
  • celle des pages "simples" qui sont des pages de validation d'action sur le site.
Chacune de ces deux structures sont décomposées en trois conteneur :
  • header: en-tête
  • div #page ou div #cadre : corps principal de la page
  • footer : pied de page
L'ensemble du contenu du site (a part les pages "simples") sont include dans la page index.php.La structure 3 colonnes de index.php comprend :
  • colonne 1 : le menu public et le menu membre
  • colonne 2 : conteneur des différentes pages du site
  • colonne 3 : conteneur de connexion et des réunions à venir / derniers membres inscrits
Tous les fichiers include dans la colonne 2 de index.php ont leur nom terminant par .inc. Les autres sont alors des pages "simples".
Tous les fichiers php sont fait de la même façon: en premier toutes les fonctions php nécessaires et ensuite la création de la page html avec les appels de fonctions php. La grande majorité du php est sous forme de fonction.

[CSS]

1er css comprend 5 fichiers :
  • header_footer.css : définit le css de l'en-tête et du pied de page de toutes les pages du site, mais aussi d'autres éléments ayant le même style quelque soit la page.
  • nav_event.css : définit le css des colonnes 1 et 3.
  • milieu.css : définit le css de la colonne 2 (conteneur central).
  • formulaire_inscription.css : définit le css de la page du formulaire d'inscription.
  • style_simple.css : définit le css des pages qui ne comportent pas les 3 colonnes.
2eme css comprend 1 fichier:
  • mobile.css : définit le css de l'ensemble du site en adaptant le css précédent pour des tailles d'écrans inférieures à 640px de largeur donc essentiellement les mobiles/smartphones. Testé sur iPhone 3GS (iOS 5.1), Samsung Galaxy Ace

[MYSQL]

La base de donnée comprend 5 tables:
  • identifiants
    • id (int) AI : id du membre
    • login (varchar-25) : pseudonyme
    • mdp (varchar-255) : mot de passe
    • nom (varchar-25) : nom du membre
    • prenom (varchar-25) : prénom du membre
    • sexe (varchar-1) : boolean du sexe du membre
    • code (int) : code postal
    • description (text) : description du membre
    • date_inscription (date) : date d'inscription au format AAAA-MM-JJ
    • nbr_connexion (int) : nombre de fois que le membre s'est connecté sur le site
    • nbr_ecriture (int) : nombre de fois que le membre a ajouté/modifié un article/projet/réunion
    • age (int) : âge du membre
    • grade (tinyint) : grade du membre [0=membre, 10=membre du conseil d'administration, 20=vice président, 30=président]
  • articles
    • id (int) AI : id de l'article
    • titre (varchar-60) : titre de l'article
    • text (text) : texte de l'article
    • auteur (int) : id de l'auteur de l'article
    • date (date) : date d'ajout de l'article au format AAAA-MM-JJ
    • modificateur (text) : ensemble des id des personnes ayant édité l'article au format ID-AAAA-MM-JJ séparé par |
    • statut (tinyint) : boolean pour article privé ou public
  • reunions
    • id (int) AI : id de la réunion
    • titre (varchar-60) : titre de la réunion
    • description (text) : texte de la réunion
    • date (date) : date de la réunion au format AAAA-MM-JJ
    • heure (time) : heure de la réunion au format hh:mm:ss
    • participants (varchar-255): ensemble des id des personnes s'étant inscrit à la réunion séparé par |
  • projets
    • id (int) AI : id du projet
    • titre (varchar-60) : titre du projet
    • description (text) : texte du projet
    • date_cration (date) : date d'ajout du projet
    • etat (tinyint) : boolean pour projet terminé ou en cours
    • participants (varchar-255) : ensemble des id des personnes participants au projet séparé par |
  • commentaires_article
    • id (int) AI : id du commentaire
    • texte (text) : texte du commentaire
    • auteur (int) : id de l'auteur du commentaire
    • article (int) : id de l'article du commentaire
    • date (date) : date d'ajout du commentaire au format AAAA-MM-JJ
    • heure (time) : heure d'ajout du commentaire au format hh:mm:ss
Pour changer l'adresse, nom de base de donnée, user ou password, il suffit de modifier les variables de la fonction connexion() dans le fichier connexion_bdd.php.
Le fichier sql pour générer la base de donnée se trouve dans le dossier Base de donnée.

[JAVASCRIPT]

Nous avons utilisé la librairie JQuery v2.0.0. Tous le javascript est situé dans des fichiers .js dans le dossier javascript. Aucun code javascript n'est présent dans les pages htmls générées
Fichiers javascript :
  • jquery.js : JQuery v2.0.0
  • aide_titre.js : met à jour un nombre pour qu'il soit équivalent à la longeur de la chaîne de caractères tapé dans un input
  • supprimer_modif.js : ouvre une fenetre de confirmation pour la suppression / affiche les personnes ayant modifié un article si plus de 2
  • commentaire.js : ajoute un commentaire à la bdd et l'ajoute au DOM / supprime un commentaire de la bdd et le cache du DOM
